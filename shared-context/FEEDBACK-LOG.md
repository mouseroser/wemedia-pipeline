## 格式
```

```

## 通用原则（适用于所有智能体）

### 执行原则
- ✅ 知道规则就执行，不要问"要我...吗？"
- ✅ 内部安全事项直接做，外部操作才问
- ✅ 默认自动执行，不反复确认

### 职责边界
- ✅ 每个智能体只做自己的本职工作
- ✅ 不走捷径，不越位代做
- ✅ Main 是编排者，不是执行者

### 通知规范
- ✅ Agent 自推到职能群（best-effort）
- ✅ Main 补发到监控群（可靠通知）
- ✅ 所有流水线和多智能体编排都必须有通知

### 记忆习惯
- ✅ 遇到问题立即记录，不用问
- ✅ 踩坑即记，防止重复
- ✅ 主动更新 MEMORY.md 和 memory/YYYY-MM-DD.md

### 2026-03-21 MiniMax 禁止用于发布/创作
- **问题**: MiniMax 模型执行小红书发布、wemedia 创作任务严重失败（空跑、不查记忆、不验证、不切方向），浪费 2+ 小时 + 重复发布 + 数据损失
- **影响**: main, wemedia, 所有涉及外部发布的 agent
- **纠正**: 小红书发布、CDP 自动化、wemedia 创作、多步骤外部操作禁止使用 MiniMax。必须用 Opus、Sonnet 或 GPT
- **状态**: ✅ 已修复（MEMORY.md、各 agent 配置已更新）

### 2026-03-21 执行前必须查记忆
- **问题**: 执行发布/生图等多步骤任务前不查已有踩坑记录，导致反复踩同一个坑
- **影响**: 所有 agent
- **纠正**: 发布/生图/多步骤任务执行前，先 `memory_recall` 查已知坑位，对照 checklist 执行
- **状态**: ✅ 已记录

### 2026-03-21 卡点处理：禁止停顿等待
- **问题**: 遇到障碍时停在解释/等待晨星提醒，而非自主解决
- **影响**: main（最严重），所有 agent

- **状态**: ⚠️ 进行中（需在后续任务中验证改善）
