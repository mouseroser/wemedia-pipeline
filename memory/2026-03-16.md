## 教训

1. 旧 PR 如果 base 指向过时分支（如 `main-legacy`），继续修会越来越脏，最好的处理方式通常是：基于当前 upstream 默认分支重建一个干净 PR，再关闭旧 PR。
2. LanceDB 底层删除记录后，OpenClaw 插件在热重载下可能仍保留旧 snapshot；要让检索结果真正刷新，必须完整 `openclaw gateway restart`。
3. auto-capture 噪音不能只靠事后手工删除，必须在审计链路里显式检查 `System:` / `Conversation info` 这类系统包络文本。
